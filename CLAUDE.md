# Goals

Standing goals for this repo. Honor them on every change; when a change trades one
off against another, say so.

1. **Faster and better terminal experience — especially keyboard navigation.** Not
   just speed (fewer keystrokes, modal/vi editing, frecency jumps) but a genuinely
   nicer workflow — better ergonomics and tools.

2. **Stay simple. Start small, grow only when a real need appears.** Prefer built-in
   features and small, well-commented config over plugins and abstractions. Don't add
   anything speculatively.

3. **One experience on both Windows and Pop!_OS.** Keybindings, theme, prompt, and
   workflow should feel identical on both. Share a single config wherever possible;
   only fork where the OS forces it. When you change a config, update both install
   scripts and verify the cross-platform path.

4. **Good AI-assisted coding and code review.** Keep the editing tools coherent with
   the goals above and easy to drive from the keyboard.

5. **Teach as we go.** Explain the why behind changes, and when we hit a natural
   point, suggest the next upgrade or skill to learn rather than just doing it
   silently.
